# Amply — Product

> 같이 듣는 순간, 방이 된다.

## 한 줄 정의

소셜 뮤직 룸. 사람들이 모여 한 명이 DJ가 되어 음악을 트는 작은 클럽.

## 영감

- **Turntable.fm** — DJ 큐에 줄 서서 한 곡씩 틀던 그 의식
- **Plug.dj** — 채팅 + 큐 + 좋아요/싫어요
- **Boiler Room / Resident Advisor** — 무드와 비주얼 톤

## 타겟 사용자

- 깊은 밤 누군가와 같이 음악 듣고 싶은 사람
- 친구 몇 명과 비공개 라운지를 차리고 싶은 사람
- DJ인 척 해보고 싶은 사람 (회원가입 없이)

## 핵심 원칙

1. **회원가입 없는 진입.** 닉네임 한 줄로 입장.
2. **한 명이 틀고 모두가 듣는다.** 동시 재생 X. 명확한 차례.
3. **로테이션이 의식이다.** 곡이 끝나면 자연스럽게 다음 사람.
4. **음악은 YouTube에 위탁.** 우리는 큐와 사회성만 만든다.
5. **MVP는 동기화하지 않는다.** 시간 정확도보다 함께라는 감각.

## 핵심 기능 (MVP)

| 영역 | 기능 |
| --- | --- |
| 입장 | 닉네임 게스트 / 이메일 매직링크 |
| 방 | 생성, 공개/비공개 토글, 슬러그 URL |
| 재생 | YouTube IFrame, 현재 DJ 1명, "곡 추가" → 즉시 재생 |
| 큐 | DJ 줄서기, 종료 시 head 로테이션, 스킵 |
| 채팅 | Postgres Changes 기반 실시간 |
| 반응 | 좋아요/싫어요 토글 (RPC) |
| 접속자 | Supabase Presence |

## MVP에서 제외

- 정밀 시간 동기화 (모두 시작 시점부터 자체 재생)
- 내 트랙 라이브러리 (DJ는 한 곡씩만 즉시 추가)
- 친구/팔로우 / 알림 / DM
- 모바일 앱 / PWA
- Spotify / Apple Music
